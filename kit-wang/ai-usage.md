I used AI tools throughout the project to bounce around design ideas, make changes on the ZenFS fork more quickly, and figure out the environment setup for testing. A lot of the work involved piecing together scattered documentation on RocksDB, ZenFS, emulated ZNS devices, and `db_bench`, so it was useful to have something that could search broadly and help interpret what different commands or build options were actually doing. One of the biggest engineering challenges were silent failures since many things appeared to run successfully even when the benchmark was not actually using the intended ZenFS path, optimization flags, ZNS device, or reclaim logic. Because of that, a lot of the project became about adding sanity checks, reading logs carefully, and verifying device state to make sure the results reflected the system I meant to test. Besides using agents for much of the engineering parts, models were pretty helpful in providing additional optimization ideas and feedback when I asked them to analyze the results from my experiments.
