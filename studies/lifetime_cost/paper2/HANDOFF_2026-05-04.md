## What pick-up needs to do (Phase 4c, ~1 focused session)

The `SchedulerOutput.block_table_filter_ops: dict[req_id, list[int]]`
field exists. The plumbing to populate + consume it is the remaining

**Three additions to make Phase 4c work end-to-end:**

1. **Scheduler populator** (in `external/memento/vllm/vllm/v1/core/sched/scheduler.py`):
   At each scheduler step, walk `self.requests`, and if any request has a
   non-empty `masked_block_ids`, populate `block_table_filter_ops[req_id]
   = list(masked_block_ids)`. Idempotent — sending the same set every
   step is a no-op when the worker has already filtered. Feed into
   `SchedulerOutput`.

2. **Worker consumer** (in `external/memento/vllm/vllm/v1/worker/gpu_model_runner.py`,
   right after `block_table_truncations` handler around line 893):
   ```python
   if scheduler_output.block_table_filter_ops:
       for req_id, block_ids_to_remove in scheduler_output.block_table_filter_ops.items():
           req_index = self.input_batch.req_id_to_index.get(req_id)
           if req_index is None:
               continue
           # For each kv_cache_group, scan block_table.np[req_index] and
           # compact-out matching block IDs. Update num_blocks_per_row and
           # input_batch.compacted_tokens_cpu (each removed block = -page_size
           # tokens) so seq_lens recomputation gives the right value.
           ...
   ```
   The actual filter logic: `mask = ~np.isin(row[:n], to_remove); row[:mask.sum()] = row[:n][mask]`.
   Subtle: the persistent `block_table.np` modification means future steps
   see the filtered version — correct for masking semantics, but recall
   needs INSERT-back (Phase 4d) to undo.

3. **GPU-side smoke test** (`paper2/tests/smoke_v4_mask.py`): build a
   small MementoVLLMModel with `attention_mask_mode=True`, fire a
   compaction, then run another chat and check (via debug print) that the
   masked block_ids are NOT in `paged_kv_indices`. The model's response
   should differ from `attention_mask_mode=False` (since attention now
   actually skips obs).

## Why this design works (RoPE-correctness sanity check)

When we filter block_table to skip obs entries:
- The kernel reads K, V from physical blocks at the indices listed in
  `paged_kv_indices` (built from filtered block_table)
- Each K vector has RoPE rotation `R(pos_k)` baked in — encoded at
  K's ORIGINAL logical position when the obs was prefilled
- The kernel computes `Q^T @ R(pos_k) @ K` which evaluates as a function
  of relative position `(pos_q - pos_k)` regardless of K's physical slot
- Result: filtering block_table doesn't break RoPE — K's content
  carries its true position via baked-in rotation
- Filtered positions are simply invisible to attention (same effect as
  Memento's physical compaction), without the cost of moving KV bytes
