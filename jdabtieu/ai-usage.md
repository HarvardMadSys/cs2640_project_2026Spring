I used AI to:
- Help find interesting papers to look at (as a supplement to Google search)
- Write the various scripts
- Generate an initial paper outline

However, I was involved in each step. Most of the sources I ended up using in the final
paper were sources I found myself, but the AI was useful in finding a few key papers
that server as good jumping off points. I proofread every script that was generated and
manually fixed a number of bugs. After the LLM generated a paper outline, the drafting and
writing was done mostly by me, with some consultation for clarity/grammar edits.

I learned that putting LLMs on maximum thinking mode can be quite useful in combining
vast amounts of information into a script or list of papers/articles to consult. I was
surprised by how long these run times took (up to 30 minutes) and also some of the really
obvious bugs it wrote. It's like with autonomous vehicles: the AI might create less bugs
than a human programmer, but some of the bugs it writes are bugs that most humans
would not write.

The biggest tip is that it's a good idea to have a solid mental model for what the code
should look like at a high level so you can give the AI as much detail as possible. This
prevents it from doing things you don't want or that aren't helpful.
